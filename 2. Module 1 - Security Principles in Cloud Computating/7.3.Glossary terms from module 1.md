## ???? English Version

### ?? Core Terminology

| Term | Definition | Practical Example |
| :--- | :--- | :--- |
| **Bucket** | A virtual container that holds objects. | Storing company logos and PDF invoices in Google Cloud Storage. |
| **Cloud Computing** | Using on-demand computing resources as services over the internet. | Using Gmail or Netflix instead of hosting your own email or video server. |
| **Cloud Cybersecurity** | Ensuring CIA (Confidentiality, Integrity, Availability) of cloud data. | Setting up Multi-Factor Authentication (MFA) to prevent unauthorized logins. |
| **Cloud Data Storage** | Off-site solution to keep, access, and maintain digital data. | Backing up your local database to an off-site cloud server for disaster recovery. |
| **Compute** | Computation performed by a physical computer in a remote environment. | Running a Python script on a virtual machine in the cloud. |
| **Container** | Software package with only necessary components for an application. | A Docker container running a specific web server version, isolated from the OS. |
| **Data Center** | Physical building storing servers and computer systems. | A massive Google-owned facility in Iowa housing thousands of physical servers. |
| **Digital Transformation** | Modernizing services and relationships using new technologies. | A bank moving from physical branch-only services to a 100% mobile app model. |
| **Ephemerality** | The concept that things only exist for a short amount of time. | A Cloud Function that runs for 2 seconds to resize an image and then disappears. |
| **Failure Domain** | A resource that can fail without impacting the availability of others. | A single server rack failing while the rest of the data center remains online. |
| **Hybrid Cloud** | A mix of public and private cloud models. | A hospital keeping sensitive patient records on-site but using the public cloud for web apps. |
| **Hypervisor** | Abstraction layer between physical hardware and Virtual Machines. | Software like VMware or KVM that allows one physical server to run 10 different OS. |
| **Immutability** | Being unable to change an object after it is created. | A "Write Once, Read Many" (WORM) policy on logs so they cannot be altered by hackers. |
| **Instance** | A server resource that runs workloads in the cloud. | Creating a "Compute Engine" instance to host a new website. |
| **Latency** | Time taken for data to travel from one location to another. | The "lag" you feel during a video call when the connection is slow. |
| **Multicloud** | Strategy of using more than one cloud service provider. | Using AWS for databases and Google Cloud for Machine Learning tools. |
| **Multi-tenant** | Environment where resources are shared among multiple users. | An apartment building where different families (tenants) share the same foundation/pipes. |
| **On-premises** | Infrastructure located in an organization’s own physical office. | A server room located in the basement of a company's headquarters. |
| **Private Cloud** | Resources dedicated to a single organization, often on-site. | A government agency running its own private cloud for high-security data. |
| **Public Cloud** | Resources shared over the internet by a provider (CSP). | Using Google Cloud Platform or Microsoft Azure. |
| **Redundancy** | Having multiple copies of data in different locations. | Saving the same file in two different regions to prevent data loss. |
| **Region** | A group of zones within a specific geographic area. | `us-east1` (South Carolina) or `southamerica-east1` (São Paulo). |
| **Repository** | Centralized place to store, download, and share data. | Your GitHub profile where you store your study Markdowns. |
| **Resiliency** | Ability to prepare for, respond to, and recover from disruptions. | An app that automatically restarts on a new server if the current one crashes. |
| **Single-tenant** | Environment where resources are dedicated to one user. | A private house where only one family uses all the resources. |
| **Structured Data** | Data organized in a fixed format (rows/columns). | An Excel spreadsheet or a SQL database table. |
| **Unstructured Data** | Data not organized in a predefined way. | A collection of various PDF files, JPG images, and video recordings. |
| **Virtualization** | Technology that creates virtual versions of physical infrastructure. | Partitioning one physical 64GB RAM server into four 16GB virtual servers. |
| **Zone** | The collective number of data centers in a specific area. | `us-central1-a` (a specific deployment area within a region). |

## ???? Versão em Português

### ?? Terminologia Essencial

| Termo | Definição | Exemplo Prático |
| :--- | :--- | :--- |
| **Bucket** | Um container virtual que armazena objetos. | Guardar logotipos da empresa e faturas em PDF no Google Cloud Storage. |
| **Cloud Computing** | Uso de recursos computacionais sob demanda via internet. | Usar o Gmail ou Netflix em vez de manter seu próprio servidor de e-mail ou vídeo. |
| **Cloud Cybersecurity** | Garantir a tríade CID (Confidencialidade, Integridade, Disponibilidade). | Configurar Autenticação de Dois Fatores (MFA) para evitar logins não autorizados. |
| **Cloud Data Storage** | Solução fora do local para manter e acessar dados digitais. | Fazer backup do seu banco de dados local em um servidor na nuvem para recuperação de desastres. |
| **Compute (Computação)** | Processamento realizado por um computador remoto. | Rodar um script Python em uma máquina virtual na nuvem. |
| **Container** | Pacote de software com apenas o necessário para rodar um app. | Um container Docker rodando uma versão específica de um site, isolado do sistema. |
| **Data Center** | Prédio físico que armazena servidores e sistemas. | Uma instalação gigante do Google que abriga milhares de servidores físicos. |
| **Digital Transformation** | Modernizar serviços e relações usando novas tecnologias. | Um banco migrando do atendimento físico para um modelo 100% digital via app. |
| **Ephemerality** | Conceito de que algo existe apenas por um curto período. | Uma Cloud Function que dura 2 segundos para processar uma foto e depois é excluída. |
| **Failure Domain** | Recurso que pode falhar sem afetar a disponibilidade de outros. | Um rack de servidores que desliga, mas o restante do data center continua operando. |
| **Hybrid Cloud** | Combinação de modelos de nuvem pública e privada. | Um hospital que mantém fichas de pacientes localmente, mas usa a nuvem para o site. |
| **Hypervisor** | Camada entre o hardware físico e as Máquinas Virtuais (VMs). | Softwares como VMware que permitem um servidor físico rodar 10 sistemas diferentes. |
| **Immutability** | Incapacidade de alterar um objeto após sua criação. | Logs de segurança configurados para "apenas leitura", impedindo que hackers apaguem rastros. |
| **Instance (Instância)** | Um recurso de servidor que executa cargas de trabalho. | Criar uma instância no "Compute Engine" para hospedar um novo site. |
| **Latency (Latência)** | Tempo que o dado leva para viajar de um local a outro. | O "atraso" que você sente em uma chamada de vídeo quando a internet está lenta. |
| **Multicloud** | Estratégia de usar mais de um provedor de nuvem. | Usar AWS para o banco de dados e Google Cloud para ferramentas de IA. |
| **Multi-tenant** | Ambiente onde recursos são compartilhados entre vários usuários. | Um prédio de apartamentos onde várias famílias compartilham a mesma estrutura/canos. |
| **On-premises** | Infraestrutura localizada fisicamente na sede da empresa. | Uma sala de servidores localizada no subsolo do prédio de uma empresa. |
| **Private Cloud** | Recursos dedicados a uma única organização. | Uma agência do governo rodando sua própria nuvem privada para dados ultra-secretos. |
| **Public Cloud** | Recursos compartilhados via internet por um provedor (CSP). | Usar serviços como Google Cloud Platform ou Microsoft Azure. |
| **Redundancy** | Prática de ter várias cópias de dados em locais diferentes. | Salvar o mesmo arquivo em duas regiões diferentes para evitar perda de dados. |
| **Region** | Grupo de zonas em uma área geográfica específica. | `us-east1` (Carolina do Sul) ou `southamerica-east1` (São Paulo). |
| **Repository** | Local centralizado para armazenar e compartilhar dados. | Seu perfil no GitHub onde você guarda seus estudos em Markdown. |
| **Resiliency** | Capacidade de se preparar e recuperar de interrupções. | Um app que reinicia automaticamente em outro servidor se o atual falhar. |
| **Single-tenant** | Ambiente onde recursos são dedicados a um único usuário. | Uma casa privativa onde apenas uma família utiliza todos os recursos. |
| **Structured Data** | Dados organizados em formato fixo (linhas/colunas). | Uma planilha de Excel ou uma tabela de banco de dados SQL. |
| **Unstructured Data** | Dados não organizados de forma predefinida. | Uma pasta com arquivos PDF, imagens JPG e gravações de vídeo. |
| **Virtualization** | Tecnologia que cria versões virtuais da infraestrutura física. | Dividir um servidor físico de 64GB de RAM em quatro servidores virtuais de 16GB. |
| **Zone** | Coletivo de data centers em uma área específica. | `us-central1-a` (uma área específica dentro de uma região). |
