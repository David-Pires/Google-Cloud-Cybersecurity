It is a security model based on three core principles that must be protected to ensure data safety.

#### 1. Confidentiality
Ensures that data is accessible only to authorized users.
* **In Google Cloud:** Handled by **IAM (Identity and Access Management)** and **Encryption** (Cloud KMS).
* **Attack Example:** A data breach where personal emails are exposed.

#### 2. Integrity
Ensures that data is accurate, complete, and has not been modified by unauthorized parties.
* **In Google Cloud:** Handled by **Cloud Storage Object Versioning** and **Digital Signatures**.
* **Attack Example:** A hacker changing the value of a financial transaction in a database.

#### 3. Availability
Ensures that data and resources are accessible when needed by authorized users.
* **In Google Cloud:** Handled by **Redundancy**, **Load Balancing**, and **Regions/Zones**.
* **Attack Example:** A **DDoS attack** that takes a website offline.
