### ðŸ¤ What is Shared Responsibility?
In the cloud, security is a partnership. Responsibilities shift depending on which service model you use (IaaS, PaaS, or SaaS).

* **Security OF the Cloud:** The CSP (Google Cloud) is responsible for protecting the global infrastructure (physical data centers, hardware, and the software that runs the cloud).
* **Security IN the Cloud:** The Customer is responsible for protecting what they put *inside* the cloud (data, user access, and application configurations).



### ðŸ—ï¸ Responsibility by Service Model

1. **IaaS (Infrastructure as a Service):** The customer has the **most** responsibility (OS patches, apps, data, network config).
2. **PaaS (Platform as a Service):** Responsibility is shared. The CSP manages the OS and runtime; the customer manages the code and data.
3. **SaaS (Software as a Service):** The customer has the **least** responsibility, focusing mainly on data and access management (IAM).
