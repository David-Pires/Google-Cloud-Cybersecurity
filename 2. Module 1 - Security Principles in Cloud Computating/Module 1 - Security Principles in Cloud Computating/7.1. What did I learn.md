### ğŸ—ï¸ Cloud Infrastructure vs. On-Premises
Cloud infrastructure refers to the hardware and software components needed to support computing requirements.
* **On-Premises:** The organization owns, manages, and maintains all physical hardware in their own data centers.
* **Cloud:** The infrastructure is managed by a Cloud Service Provider (CSP). This shift moves costs from CapEx (Capital Expenditure) to OpEx (Operating Expenditure).



### ğŸ“¦ Containers in the Cloud
Containers are lightweight, standalone packages that contain everything needed to run an application (code, runtime, system tools, libraries).
* **Portability:** They run the same way regardless of the environment (development, test, or production).
* **Efficiency:** Unlike Virtual Machines (VMs), containers share the host's Operating System (OS) kernel, making them much faster and lighter.
* **Security Benefit:** Provides isolation between applications; if one container is compromised, the breach is contained within that specific environment.

### ğŸ’¾ Cloud Storage Options
Understanding how data is stored is crucial for security and performance:
1. **File Storage:** Organized in a hierarchy of files and folders (e.g., Google Drive, SMB). Best for shared access to files.
2. **Block Storage:** Data is split into fixed-size blocks. Highly efficient and used for databases or OS drives (e.g., Persistent Disks).
3. **Object Storage:** Data is stored as individual objects with metadata and a unique identifier. Highly scalable (e.g., Google Cloud Storage).
