


# 🛡️ Data States & Protection Review | Revisão de Estados e Proteção

### 1. Data in Use (Dados em Uso) | Questão 1

**🇺🇸 English:** When a user is actively reviewing, accessing, or processing information (like detailed account info on a portal), the data is **in use**. This is the state where data is loaded into the system's memory (RAM) for interaction.

**🇧🇷 Português:** Quando um usuário está revisando, acessando ou processando informações ativamente (como detalhes de conta em um portal), os dados estão **em uso**. Este é o estado em que os dados são carregados na memória do sistema (RAM) para interação.

---

### 2. Data in Transit (Dados em Trânsito) | Questão 2

**🇺🇸 English:** Data moving between a client (mobile device) and a server (banking host) is **in transit**. The primary protection for this state is **TLS**, which creates an encrypted tunnel to prevent interception during the transfer across networks.

**🇧🇷 Português:** Dados que se movem entre um cliente (dispositivo móvel) e um servidor (hospedeiro bancário) estão **em trânsito**. A principal proteção para este estado é o **TLS**, que cria um túnel criptografado para evitar a interceptação durante a transferência pelas redes.

---

### 3. Confidential Computing & TEE | Questão 3

**🇺🇸 English:** To protect **data in use**, we implement **Confidential Computing**. This technology uses **Trusted Execution Environments (TEEs)**—hardware-based isolated enclaves—to ensure that data remains encrypted even while being processed by the CPU.

**🇧🇷 Português:** Para proteger **dados em uso**, implementamos a **Computação Confidencial**. Esta tecnologia utiliza **Ambientes de Execução Confiáveis (TEEs)** — enclaves isolados baseados em hardware — para garantir que os dados permaneçam criptografados mesmo enquanto são processados pela CPU.

---

### 4. Key Rotation & Data Chunking | Questão 4

**🇺🇸 English:** For **data at rest**, a robust security practice is breaking data into chunks. When a chunk is updated, it receives a **new encryption key**. This process of regular key updates (rotation) significantly reduces the impact of a potential key compromise.

**🇧🇷 Português:** Para **dados em repouso**, uma prática de segurança robusta é dividir os dados em pedaços (*chunks*). Quando um pedaço é atualizado, ele recebe uma **nova chave de criptografia**. Este processo de atualizações regulares de chaves (rotação) reduz significativamente o impacto de um eventual comprometimento de chave.

---

## 🛡️ Analyst's Insight | Insight do Analista

**🇺🇸 English:**
The "Takeaway" from this assessment is the importance of **Defense in Depth**. Notice that for each state, the vulnerability and the tool change. Protecting data on a disk (At Rest) does not protect it while it's being sent to the user (In Transit). You must apply all three layers to ensure a truly secure cloud environment.

**🇧🇷 Português:**
*A lição principal desta avaliação é a importância da **Defesa em Profundidade**. Note que para cada estado, a vulnerabilidade e a ferramenta mudam. Proteger os dados em um disco (Em Repouso) não os protege enquanto estão sendo enviados ao usuário (Em Trânsito). Você deve aplicar todas as três camadas para garantir um ambiente de nuvem realmente seguro.*
