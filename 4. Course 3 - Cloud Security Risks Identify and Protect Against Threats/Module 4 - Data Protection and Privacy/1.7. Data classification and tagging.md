

# 🏷️ Data Classification & Tagging | Classificação e Etiquetagem de Dados

### 1. Sensitivity Levels | Níveis de Sensibilidade

**🇺🇸 English:** Data classification is the process of analyzing data to determine its sensitivity and value. Organizations categorize assets into three main levels to prioritize security efforts:

* **Low Sensitivity:** Public information (e.g., a public blog).
* **Medium Sensitivity:** Internal data not meant for the public (e.g., internal memos or project outlines).
* **High Sensitivity:** Critical data like **PII** (Personally Identifiable Information), financial records, or trade secrets. Leakage leads to legal fines and reputational damage.

**🇧🇷 Português:** A classificação de dados é o processo de analisar os dados para determinar sua sensibilidade e valor. As organizações categorizam os ativos em três níveis principais para priorizar os esforços de segurança:

* **Baixa Sensibilidade:** Informações públicas (ex: um blog público).
* **Média Sensibilidade:** Dados internos não destinados ao público (ex: memorandos internos ou esboços de projetos).
* **Alta Sensibilidade:** Dados críticos como **PII** (Informações de Identificação Pessoal), registros financeiros ou segredos comerciais. O vazamento acarreta multas legais e danos à reputação.

---

### 2. The Power of Tags | O Poder das Tags (Etiquetas)

**🇺🇸 English:** **Tags** are custom metadata fields attached to data entries to provide context. They act as "flags" for security controls. For example, a security system can automatically encrypt any file that has a tag labeled `PII: True`. Without tags, security controls wouldn't know which assets require the highest protection.

**🇧🇷 Português:** **Tags** são campos de metadados personalizados anexados às entradas de dados para fornecer contexto. Elas atuam como "sinalizadores" para os controles de segurança. Por exemplo, um sistema de segurança pode criptografar automaticamente qualquer arquivo que tenha uma tag rotulada como `PII: True`. Sem as tags, os controles de segurança não saberiam quais ativos exigem a maior proteção.

---

### 3. Practical Example: PII Tag | Exemplo Prático: Tag de PII

**🇺🇸 English:** A tag often uses **Boolean (BOOL)** fields (True/False) and key-value pairs to describe the data. Look at this structure:

* **Label:** `PII`
* **Metadata Pair 1:** `Value: True` (Confirms PII is present).
* **Metadata Pair 2:** `Type: SSN` (Identifies it as a Social Security Number).

**🇧🇷 Português:** Uma tag geralmente usa campos **Booleanos (BOOL)** (Verdadeiro/Falso) e pares chave-valor para descrever os dados. Observe esta estrutura:

* **Rótulo (Label):** `PII`
* **Par de Metadados 1:** `Value: True` (Confirma que há PII presente).
* **Par de Metadados 2:** `Type: SSN` (Identifica como um número de CPF/Seguro Social).

---

### 4. Tag Templates & IAM | Modelos de Tags e IAM

**🇺🇸 English:** To scale tagging, organizations use **Tag Templates**—reusable structures for rapid classification. Access to these templates is managed via **IAM Roles**, ensuring that only authorized personnel can create or modify the classification of an asset.

**🇧🇷 Português:** Para escalar a etiquetagem, as organizações usam **Modelos de Tags (Tag Templates)** — estruturas reutilizáveis para classificação rápida. O acesso a esses modelos é gerenciado via **Regras de IAM**, garantindo que apenas pessoal autorizado possa criar ou modificar a classificação de um ativo.

## 🛡️ Analyst's Insight | Insight do Analista

**🇺🇸 English:**
Tagging is the foundation of **Automation**. As a cloud security professional, your goal is to create policies like: *"If an object is tagged as 'High Sensitivity', prevent it from being moved to a public storage bucket."* This removes human error from the equation and ensures that even if a developer makes a mistake, the system automatically protects the sensitive data.

**🇧🇷 Português:**
*A etiquetagem é a base da **Automação**. Como profissional de segurança em nuvem, seu objetivo é criar políticas como: "Se um objeto for etiquetado como 'Alta Sensibilidade', impeça que ele seja movido para um repositório de armazenamento público." Isso remove o erro humano da equação e garante que, mesmo que um desenvolvedor cometa um erro, o sistema proteja automaticamente os dados sensíveis.*

---

### 🚀 Key Takeaway

**"Classification tells you WHAT to protect; Tags tell the SYSTEM how to find it."**
*(A classificação diz a VOCÊ o que proteger; as Tags dizem ao SISTEMA como encontrar.)*
