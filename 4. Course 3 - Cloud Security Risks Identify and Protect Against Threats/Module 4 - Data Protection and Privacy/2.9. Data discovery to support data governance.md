


# 🔍 Data Discovery | Descoberta de Dados

### 1. O que é Descoberta de Dados? | What is Data Discovery?

**🇺🇸 English:** Data discovery is the process of searching, identifying, and analyzing large volumes of data to uncover patterns and sensitive assets. It supports **Data Governance** by identifying which data exists and **Data Sovereignty** by flagging information that must follow specific geographic laws. It ensures **Data Transparency**, making information available and searchable for compliance and security.

**🇧🇷 Português:** A descoberta de dados é o processo de pesquisar, identificar e analisar grandes volumes de dados para descobrir padrões e ativos sensíveis. Ela apoia a **Governança de Dados** ao identificar quais dados existem e a **Soberania de Dados** ao sinalizar informações que devem seguir leis geográficas específicas. Isso garante a **Transparência de Dados**, tornando a informação disponível e pesquisável para conformidade e segurança.

---

### 2. Descoberta de Dados Sensíveis | Sensitive Data Discovery

**🇺🇸 English:** This specialized form of discovery focuses on finding confidential and regulated information, such as:

* **Payment Data:** Credit card numbers.
* **PHI (Protected Health Information):** Medical records.
* **Intellectual Property:** Trade secrets and proprietary code.
It is vital to know exactly **what** the data is and **where** it is located to apply the correct security controls.

**🇧🇷 Português:** Esta forma especializada de descoberta foca em encontrar informações confidenciais e regulamentadas, tais como:

* **Dados de Pagamento:** Números de cartão de crédito.
* **PHI (Informações de Saúde Protegidas):** Registros médicos.
* **Propriedade Intelectual:** Segredos comerciais e código proprietário.
É vital saber exatamente **o que** é o dado e **onde** ele está localizado para aplicar os controles de segurança corretos.

---

### 3. O Motor de DLP (Data Loss Prevention) | The DLP Engine

**🇺🇸 English:** A **Data Loss Prevention (DLP)** engine is the primary tool for sensitive data discovery. It functions at various stages of the data lifecycle to:

* **Inspect:** Scan data for sensitive patterns (like SSN formats).
* **Mask/Redact:** Obscure or remove sensitive data once discovered.
* **Filter:** Prevent sensitive data from being moved to insecure or unrestricted areas.

**🇧🇷 Português:** Um motor de **Prevenção de Perda de Dados (DLP)** é a ferramenta principal para a descoberta de dados sensíveis. Ele funciona em vários estágios do ciclo de vida dos dados para:

* **Inspecionar:** Escanear dados em busca de padrões sensíveis (como formatos de CPF).
* **Mascarar/Redigir:** Ocultar ou remover dados sensíveis após a descoberta.
* **Filtrar:** Impedir que dados sensíveis sejam movidos para áreas inseguras ou irrestritas.

---

### 4. Validação em Migrações | Validation in Migrations

**🇺🇸 English:** During cloud migrations, data discovery acts as a validator. It checks if data is being moved into properly secured environments and ensures there are no mistakes in **Classification** or placement. This prevents regulated data from accidentally landing in an environment without proper controls.

**🇧🇷 Português:** Durante as migrações para a nuvem, a descoberta de dados atua como um validador. Ela verifica se os dados estão sendo movidos para ambientes devidamente protegidos e garante que não haja erros na **Classificação** ou no posicionamento. Isso evita que dados regulamentados parem acidentalmente em um ambiente sem os controles adequados.

## 🛡️ Analyst's Insight | Insight do Analista

**🇺🇸 English:**
Data Discovery is your "Reality Check". Developers might think they aren't storing PII in a specific database, but a DLP scan might find unencrypted emails or IDs. As a professional, use automated DLP scans as a **Continuous Security Control** to catch "Shadow Data"—sensitive information that exists outside of your official inventory.

**🇧🇷 Português:**
*A Descoberta de Dados é o seu "Choque de Realidade". Os desenvolvedores podem achar que não estão armazenando PII em um banco de dados específico, mas um scan de DLP pode encontrar e-mails ou IDs não criptografados. Como profissional, use scans de DLP automatizados como um **Controle de Segurança Contínuo** para capturar "Dados Sombrios" (Shadow Data) — informações sensíveis que existem fora do seu inventário oficial.*
