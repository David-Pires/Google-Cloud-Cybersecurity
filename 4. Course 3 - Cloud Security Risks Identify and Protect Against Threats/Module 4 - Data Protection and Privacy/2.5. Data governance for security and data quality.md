

# 🏛️ Data Governance | Governança de Dados

### 1. O que é Governança de Dados? | What is Data Governance?

**🇺🇸 English:** Data Governance is a set of processes, roles, and technologies that ensure an organization's data is private, accurate, secure, available, and usable. A key figure in this process is the **Data Steward**, a subject matter expert responsible for collecting, managing, and preserving the quality of data assets throughout their lifecycle.

**🇧🇷 Português:** A Governança de Dados é um conjunto de processos, funções e tecnologias que garantem que os dados de uma organização sejam privados, precisos, seguros, disponíveis e utilizáveis. Uma figura central neste processo é o **Data Steward** (Curador de Dados), um especialista responsável por coletar, gerenciar e preservar a qualidade dos ativos de dados ao longo de todo o seu ciclo de vida.

---

### 2. As Seis Dimensões da Qualidade | The Six Dimensions of Data Quality

Para que o dado tenha valor para a segurança e para o negócio, ele deve atender a seis critérios:

| 🇺🇸 Dimension | 🇧🇷 Dimensão | 💡 Meaning / Significado |
| --- | --- | --- |
| **Accuracy** | **Precisão** | Data matches a verifiable source. / O dado corresponde a uma fonte verificável. |
| **Completeness** | **Completude** | All required fields have information. / Todos os campos obrigatórios possuem informação. |
| **Consistency** | **Consistência** | Values match across multiple databases. / Os valores coincidem em múltiplos bancos de dados. |
| **Timeliness** | **Pontualidade** | Data is available when needed (e.g., for an audit). / O dado está disponível quando necessário (ex: para uma auditoria). |
| **Validity** | **Validade** | Data aligns with specific requirements (e.g., card number length). / O dado segue requisitos específicos (ex: tamanho do número do cartão). |
| **Uniqueness** | **Unicidade** | Minimal overlaps or no duplicate records. / Mínimas sobreposições ou ausência de registros duplicados. |

---

### 3. Policies and Procedures | Políticas e Procedimentos

**🇺🇸 English:** A formal governance framework establishes clear policies for **access, use, storage, and disposal** of data. These policies must align with industry standards and regulatory requirements. Their goal is twofold: preventing unauthorized access (Security) while ensuring authorized users can access data when needed (Availability).

**🇧🇷 Português:** Um framework formal de governança estabelece políticas claras para o **acesso, uso, armazenamento e descarte** de dados. Essas políticas devem estar alinhadas com padrões da indústria e requisitos regulatórios. Seu objetivo é duplo: impedir o acesso não autorizado (Segurança) e, ao mesmo tempo, garantir que usuários autorizados acessem os dados quando necessário (Disponibilidade).

## 🛡️ Analyst's Insight | Insight do Analista

**🇺🇸 English:**
From a security perspective, **Consistency** and **Accuracy** are vital for Incident Response. If a user's ID is spelled differently in the HR database versus the Access Log, identifying the person behind a security alert becomes a nightmare. Data Governance fixes this by enforcing standards across the entire cloud environment.

**🇧🇷 Português:**
*Do ponto de vista da segurança, a **Consistência** e a **Precisão** são vitais para a Resposta a Incidentes. Se o ID de um usuário estiver escrito de forma diferente no banco de dados de RH em comparação ao log de acesso, identificar a pessoa por trás de um alerta de segurança torna-se um pesadelo. A Governança de Dados resolve isso impondo padrões em todo o ambiente de nuvem.*

### 🚀 Key Takeaway

**"Security locks the door, but Governance ensures the house is organized and the records are true."**
*(A segurança tranca a porta, mas a governança garante que a casa esteja organizada e que os registros sejam verdadeiros.)*
