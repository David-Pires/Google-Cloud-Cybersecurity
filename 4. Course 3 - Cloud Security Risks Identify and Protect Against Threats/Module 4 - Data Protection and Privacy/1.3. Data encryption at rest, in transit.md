




# 🛡️ Encryption for the Three States | Criptografia para os Três Estados

### 1. Data at Rest (Dados em Repouso)

**🇺🇸 English:** Data at rest refers to any data not currently being accessed, such as files on a hard drive. The global standard for protecting this state is **AES (Advanced Encryption Standard)**. A powerful technique involves breaking data into **chunks**, where each chunk is encrypted with its own **Data Encryption Key (DEK)**. This minimizes the risk because if one key is compromised, only a small part of the data is exposed.

**🇧🇷 Português:** Dados em repouso referem-se a quaisquer dados que não estão sendo acessados no momento, como arquivos em um disco rígido. O padrão global para proteger este estado é o **AES (Advanced Encryption Standard)**. Uma técnica poderosa envolve dividir os dados em **pedaços (chunks)**, onde cada pedaço é criptografado com sua própria **Chave de Criptografia de Dados (DEK)**. Isso minimiza o risco, pois se uma chave for comprometida, apenas uma pequena parte dos dados será exposta.

---

### 2. Data in Transit (Dados em Trânsito)

**🇺🇸 English:** Data in transit is information moving between applications or over the internet. To secure this, we use **TLS (Transport Layer Security)**. This protocol ensures that sensitive information, like credit card numbers or passwords sent via a banking app, remains unreadable to attackers sniffing the network.

**🇧🇷 Português:** Dados em trânsito são informações que se movem entre aplicativos ou pela internet. Para proteger isso, usamos o **TLS (Transport Layer Security)**. Este protocolo garante que informações sensíveis, como números de cartão de crédito ou senhas enviadas por um aplicativo bancário, permaneçam ilegíveis para atacantes que estejam monitorando a rede.

---

### 3. Data in Use (Dados em Uso)

**🇺🇸 English:** Data in use is being actively processed in the RAM or CPU. Protecting this state is the goal of **Confidential Computing**, which utilizes a **Trusted Execution Environment (TEE)**. A TEE is a hardware-based isolated environment that uses embedded encryption keys to prevent unauthorized access or modification of data while it is being computed.

**🇧🇷 Português:** Dados em uso estão sendo processados ativamente na RAM ou CPU. Proteger este estado é o objetivo da **Computação Confidencial**, que utiliza um **Ambiente de Execução Confiável (TEE)**. Um TEE é um ambiente isolado baseado em hardware que usa chaves de criptografia incorporadas para impedir o acesso não autorizado ou a modificação de dados enquanto eles estão sendo computados.

## 🛡️ Analyst's Insight | Insight do Analista

**🇺🇸 English:**
A common mistake is focusing only on **At Rest** and **In Transit** encryption. However, "Data in Use" is a major target for advanced memory-scraping malware. As a professional, when you choose a cloud instance, check if it supports **Confidential VMs**. This adds that final layer of hardware-level protection that makes it impossible even for the cloud provider's administrators to see your data while it's being processed.

**🇧🇷 Português:**
*Um erro comum é focar apenas na criptografia **Em Repouso** e **Em Trânsito**. No entanto, o "Dado em Uso" é um alvo principal para malwares avançados de captura de memória. Como profissional, ao escolher uma instância de nuvem, verifique se ela suporta **VMs Confidenciais**. Isso adiciona aquela camada final de proteção a nível de hardware que torna impossível até para os administradores do provedor de nuvem verem seus dados enquanto eles estão sendo processados.*

---

### 🚀 Key Takeaway

**"Encryption is the lock, but the key management is the security. Rotate your DEKs and use TEEs to ensure end-to-end privacy."**
*(A criptografia é a fechadura, mas a gestão de chaves é a segurança. Rotacione suas DEKs e use TEEs para garantir a privacidade de ponta a ponta.)*
