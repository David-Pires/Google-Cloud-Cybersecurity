



# 🏷️ Data Classification Review | Revisão de Classificação de Dados

### 1. The Role of Tags | O Papel das Tags (Questões 1 e 2)

**🇺🇸 English:** Tags are **custom metadata fields** assigned to data entries to provide context. They are crucial because they enable better organization of sensitive data and act as beacons, helping security assets (like automated encryption tools or scanners) find exactly which data needs protection.

**🇧🇷 Português:** Tags são **campos de metadados personalizados** atribuídos a entradas de dados para fornecer contexto. Elas são cruciais porque permitem uma melhor organização de dados sensíveis e funcionam como sinalizadores, ajudando ativos de segurança (como ferramentas de criptografia automática) a encontrar exatamente quais dados precisam de proteção.

---

### 2. Practical Application of Sensitivity Levels | Aplicação Prática dos Níveis (Questões 3, 4 e 5)

A classificação correta depende do impacto potencial de um vazamento:

* **Public (Público):**
* **🇺🇸 Use Case:** Information intended for marketing or general awareness (e.g., pricing tiers, service limits).
* **🇧🇷 Caso de Uso:** Informações destinadas a marketing ou conhecimento geral (ex: níveis de preços, limites de serviço).


* **Medium (Médio):**
* **🇺🇸 Use Case:** Internal-only communication that doesn't contain secrets or PII (e.g., a project presentation shared on a private messaging tool). It requires access control but isn't critical.
* **🇧🇷 Caso de Uso:** Comunicação apenas interna que não contém segredos ou PII (ex: uma apresentação de projeto compartilhada em ferramenta de mensagens privada). Requer controle de acesso, mas não é crítica.


* **High (Alto):**
* **🇺🇸 Use Case:** Data that includes payment details (credit cards), PII, or contact info. This requires the highest level of security to prevent identity theft and legal consequences.
* **🇧🇷 Caso de Uso:** Dados que incluem detalhes de pagamento (cartões de crédito), PII ou informações de contato. Exige o nível mais alto de segurança para evitar roubo de identidade e consequências legais.

## 🛡️ Analyst's Insight | Insight do Analista

**🇺🇸 English:**
Notice how Question 2 connects **Tags** to **Security Assets**. In a modern cloud environment, you don't manually protect every file. You set a policy: *"Every object with the tag `Sensitivity: High` must be stored in an encrypted bucket with restricted IAM access."* This makes security scalable and reduces the chance of "orphaned" sensitive data.

**🇧🇷 Português:**
*Note como a Questão 2 conecta **Tags** a **Ativos de Segurança**. Em um ambiente de nuvem moderno, você não protege cada arquivo manualmente. Você define uma política: "Todo objeto com a tag `Sensitivity: High` deve ser armazenado em um bucket criptografado com acesso IAM restrito." Isso torna a segurança escalável e reduz a chance de dados sensíveis ficarem "órfãos" (sem proteção).*
