# 🔐 Fundamentos de Criptografia de Dados | Data Encryption Basics

### 1. O Processo de Criptografia | The Encryption Process

**🇺🇸 English:** Data encryption is the process of converting human-readable **plaintext** into encoded **ciphertext** using mathematical cryptographic algorithms. This ensures that data is protected from being changed, compromised, or stolen by scrambling it into a secret code that only a unique **digital key** can unlock.

**🇧🇷 Português:** A criptografia de dados é o processo de converter **texto simples** (legível) em **texto cifrado** (codificado) usando algoritmos criptográficos matemáticos. Isso garante que os dados sejam protegidos contra alteração, comprometimento ou roubo, transformando-os em um código secreto que apenas uma **chave digital** exclusiva pode desbloquear.

---

### 2. Responsabilidade na Nuvem | Responsibility in the Cloud

**🇺🇸 English:** When using cloud services, responsibility for encryption can be **client-side** or **server-side**. In client-side encryption, the user creates and manages keys before sending data to the cloud. In server-side encryption, the cloud provider handles key management.

**🇧🇷 Português:** Ao usar serviços de nuvem, a responsabilidade pela criptografia pode ser do **lado do cliente** ou do **lado do servidor**. Na criptografia do lado do cliente, o usuário cria e gerencia as chaves antes de enviar os dados para a nuvem. Na criptografia do lado do servidor, o provedor de nuvem cuida do gerenciamento das chaves.

---

### 3. As Quatro Funções da Criptografia | The Four Functions of Encryption

A criptografia robusta deve garantir quatro pilares fundamentais:

| 🇺🇸 Function | 🇧🇷 Função | 🛡️ Purpose / Propósito |
| --- | --- | --- |
| **Confidentiality** | **Confidencialidade** | Keep contents secret. / Manter o conteúdo em segredo.

 |
| **Authenticity** | **Autenticidade** | Verify data origin. / Verificar a origem dos dados.

 |
| **Integrity** | **Integridade** | Ensure data hasn't been altered. / Garantir que os dados não foram alterados.

 |
| **Nonrepudiation** | **Não repúdio** | Prevent sender from denying origin (asymmetric only). / Impedir que o remetente negue a origem (apenas assimétrica).

 |

---

### 4. Padrões de Criptografia: AES vs. RSA | Encryption Standards

**🇺🇸 English:** **AES (Advanced Encryption Standard)** is a symmetric block cipher (128, 192, or 256 bits) used for fast encryption of everyday data like banking credentials and Wi-Fi. **RSA (Rivest-Shamir-Adleman)** is an early form of asymmetric encryption used for verifying identities and secure communications.

**🇧🇷 Português:** **AES (Advanced Encryption Standard)** é uma cifra de bloco simétrica (128, 192 ou 256 bits) usada para criptografia rápida de dados cotidianos, como credenciais bancárias e Wi-Fi. **RSA (Rivest-Shamir-Adleman)** é uma das primeiras formas de criptografia assimétrica, usada para verificar identidades e comunicações seguras.

---

### 5. Desafios e Gestão de Chaves | Challenges and Key Management

**🇺🇸 English:** Encryption faces challenges like **ransomware**, where attackers encrypt an organization's data for ransom. Secure **key management** is vital; losing keys in a disaster or attack can permanently lock an organization out of its own data.

**🇧🇷 Português:** A criptografia enfrenta desafios como o **ransomware**, onde atacantes criptografam os dados de uma organização para exigir resgate. A **gestão segura de chaves** é vital; perder as chaves em um desastre ou ataque pode bloquear permanentemente o acesso de uma organização aos seus próprios dados.

---

## 🛡️ Analyst's Insight | Insight do Analista

**🇺🇸 English:** Robust encryption relies on large key lengths to prevent **brute force attacks**, making it mathematically impossible for attackers to guess the correct sequence. As a security professional, always prioritize **256-bit AES** for data at rest whenever possible for maximum security.

**🇧🇷 Português:** A criptografia robusta depende de chaves longas para evitar **ataques de força bruta**, tornando matematicamente impossível para os atacantes adivinharem a sequência correta. Como profissional de segurança, sempre priorize o **AES de 256 bits** para dados em repouso sempre que possível para garantir o máximo de segurança.
