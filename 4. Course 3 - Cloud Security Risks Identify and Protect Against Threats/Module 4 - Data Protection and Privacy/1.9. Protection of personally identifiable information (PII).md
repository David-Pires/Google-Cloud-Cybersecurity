
# 🛡️ Proteção de Informações de Identificação Pessoal (PII)

### 1. O que é PII? | What is PII?

**🇺🇸 English:** **Personally Identifiable Information (PII)** is any data that can be used to identify a specific individual. This is considered highly sensitive data. Examples include names, home addresses, biometric records, medical history, educational data, financial information, and employment records.

**🇧🇷 Português:** **Informações de Identificação Pessoal (PII)** são quaisquer dados que possam ser usados para identificar um indivíduo específico. São considerados dados de alta sensibilidade. Exemplos incluem nomes, endereços residenciais, registros biométricos, histórico médico, dados educacionais, informações financeiras e registros de emprego.

---

### 2. O Risco da Perda de Dados | Data Loss and Sensitivity

**🇺🇸 English:** Sensitive data is information that could cause embarrassment, harm to an individual, or damage an organization's reputation if leaked. **Data loss** is the unauthorized exposure of proprietary or sensitive data through leakage or theft. Protecting this data is a primary responsibility for cloud security teams.

**🇧🇷 Português:** Dados sensíveis são informações que podem causar constrangimento, danos a um indivíduo ou prejudicar a reputação de uma organização se forem vazados. A **perda de dados** é a exposição não autorizada de dados proprietários ou sensíveis por meio de fuga ou roubo. Proteger esses dados é uma responsabilidade primordial das equipas de segurança na nuvem.

---

### 3. Estratégias de Proteção | Protection Strategies

Existem duas abordagens principais para proteger PII:

* **IAM (Identity and Access Management):** * 🇺🇸 *Control access by asking: Who is accessing the data? and Why do they need it?*
* 🇧🇷 Controlar o acesso perguntando: Quem está a aceder aos dados? e Por que precisam deles?


* **De-identification (Descaracterização):**
* 🇺🇸 *The process of removing identifying information so data cannot be linked to a specific person.*
* 🇧🇷 O processo de remover informações de identificação para que os dados não possam ser vinculados a uma pessoa específica.



---

### 4. Métodos de Descaracterização | De-identification Methods

**🇺🇸 English:** Cloud security professionals use three main methods to de-identify data:

1. **Redaction:** Deleting all or part of a sensitive value.
2. **Replacement:** Replacing a sensitive value with a generic value.
3. **Data Masking:** Replacing characters with generic symbols, like hashes or asterisks (e.g., `****-****-1234`).

**🇧🇷 Português:** Os profissionais de segurança na nuvem usam três métodos principais para descaracterizar dados:

1. **Redação (Redaction):** Excluir todo ou parte de um valor sensível.
2. **Substituição (Replacement):** Substituir um valor sensível por um valor genérico.
3. **Mascaramento de Dados (Data Masking):** Substituir caracteres por símbolos genéricos, como hashes ou asteriscos (ex: `****-****-1234`).

## 🛡️ Insight do Analista | Analyst's Insight

**🇺🇸 English:** **Data Masking** is essential for environments like "Testing" or "Staging". Developers often need real data to test applications, but they shouldn't see actual PII. By masking the database, you allow the system to function for testing purposes without ever exposing the real identities of your customers.

**🇧🇷 Português:** O **Mascaramento de Dados** é essencial para ambientes de "Teste" ou "Staging". Os desenvolvedores muitas vezes precisam de dados reais para testar aplicações, mas não devem ver PII reais. Ao mascarar a base de dados, permite que o sistema funcione para fins de teste sem nunca expor as identidades reais dos seus clientes.
