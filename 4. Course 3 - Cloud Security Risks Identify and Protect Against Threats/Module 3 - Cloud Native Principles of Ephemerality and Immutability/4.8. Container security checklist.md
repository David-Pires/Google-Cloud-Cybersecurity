# **Container security checklist**

- [x] Perform vulnerability scanning on container images.

**Explanation**: Container vulnerability found. Action: Scan SO/Kernell configurations and updates. Redeploy the container with a new version of the shared image.  

- [x] Use role-based access control to control access for user and service accounts.

**Explanation**: Trust Zero application. Give only necessary permission for the Kubernetes service account to avoid a threat vector exploitation.
