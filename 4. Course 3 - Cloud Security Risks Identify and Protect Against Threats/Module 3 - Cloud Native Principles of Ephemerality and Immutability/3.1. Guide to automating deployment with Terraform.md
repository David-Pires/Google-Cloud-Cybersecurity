# 🛠️ Practical Terraform Guide | Guia Prático de Terraform

### 🇺🇸 English

Terraform uses **HashiCorp Configuration Language (HCL)**, a declarative syntax designed to be human-readable. Instead of listing steps, you define **Blocks** and **Arguments** to describe your goal. The configuration files always end with the `.tf` extension.

### 🇧🇷 Português

*O Terraform utiliza a **HashiCorp Configuration Language (HCL)**, uma sintaxe declarativa projetada para ser legível por humanos. Em vez de listar passos, você define **Blocos** e **Argumentos** para descrever seu objetivo. Os arquivos de configuração sempre terminam com a extensão `.tf`.*

---

## 🧱 Anatomy of a `.tf` File | Anatomia de um Arquivo `.tf`

Baseado no exemplo `main.tf` da leitura:

1. **Resource Blocks (Blocos de Recurso):**
* 🇺🇸 *Containers for objects like firewalls or networks. In the example, we see `google_compute_firewall` and `google_compute_network`.*
* 🇧🇷 Contêineres para objetos como firewalls ou redes. No exemplo, vemos `google_compute_firewall` e `google_compute_network`.


2. **Labels (Rótulos):**
* 🇺🇸 *The first label is the **resource type** (provided by the CSP); the second is the **local name** you give to it (e.g., "default").*
* 🇧🇷 O primeiro rótulo é o **tipo de recurso** (fornecido pelo provedor); o segundo é o **nome local** que você dá a ele (ex: "default").


3. **Arguments (Argumentos):**
* 🇺🇸 *Key-value pairs inside the block that set specific configurations, such as `name = "test-firewall"` or allowed protocols like `icmp` or `tcp`.*
* 🇧🇷 Pares de chave-valor dentro do bloco que definem configurações específicas, como `name = "test-firewall"` ou protocolos permitidos como `icmp` ou `tcp`.

## ⌨️ Essential Commands | Comandos Essenciais

Para realizar o deploy no Google Cloud Shell, você seguirá este fluxo:

| Command / Comando | 🇺🇸 Purpose | 🇧🇷 Objetivo |
| --- | --- | --- |
| `terraform init` | Initializes the directory and downloads necessary plugins (providers). | Inicializa o diretório e baixa os plugins necessários (providers). |
| `terraform plan`* | Previews changes before they happen (The "safety check"). | Pré-visualiza as mudanças antes que ocorram (O "check de segurança"). |
| `terraform apply` | Executes the plan. You must type **"yes"** to confirm provisioning. | Executa o plano. Você deve digitar **"yes"** para confirmar o provisionamento. |

**Nota: Embora o texto foque em init e apply, o `plan` é o passo intermediário crítico para segurança mencionado em vídeos anteriores.*

## 🛡️ Analyst's Insight | Insight do Analista

**🇺🇸 English:**
Notice the connection in the code: `network = google_compute_network.default.name`. This is a **Dependency**. Terraform is smart enough to know it must create the **Network** *before* it can attach the **Firewall** to it. As a security analyst, this ensures your security controls (Firewalls) are always correctly mapped to the right infrastructure segments automatically.

**🇧🇷 Português:**
*Note a conexão no código: `network = google_compute_network.default.name`. Isso é uma **Dependência**. O Terraform é inteligente o suficiente para saber que deve criar a **Rede** *antes* de poder anexar o **Firewall** a ela. Como analista de segurança, isso garante que seus controles de segurança (Firewalls) estejam sempre mapeados corretamente para os segmentos de infraestrutura certos de forma automática.*
