

# 🏗️ IaC: Declarative vs. Imperative | Abordagens de IaC

### 🇺🇸 English

**Infrastructure as Code (IaC)** is the practice of managing and provisioning infrastructure through machine-readable definition files. The industry standard is the **Declarative Approach**, where you define the "What" (the desired end state) and the tool handles the "How".

### 🇧🇷 Português

*A **Infraestrutura como Código (IaC)** é a prática de gerenciar e provisionar infraestrutura por meio de arquivos de definição legíveis por máquina. O padrão da indústria é a **Abordagem Declarativa**, onde você define o "O quê" (o estado final desejado) e a ferramenta cuida do "Como".*

---

## ⚖️ Comparing the Approaches | Comparando as Abordagens

| Feature / Recurso | **Declarative (Ex: Terraform)** | **Imperative (Ex: Scripts Bash/CLI)** |
| --- | --- | --- |
| **Focus / Foco** | **Desired State:** Defines the outcome. / Define o resultado final. | **Step-by-Step:** Defines the commands. / Define os comandos passo a passo. |
| **State / Estado** | Tool compares current vs. desired. / Ferramenta compara o estado atual com o desejado. | You must track the order and state manually. / Você deve rastrear a ordem e o estado manualmente. |
| **Simplicity / Simplicidade** | Easier to manage over lifecycle. / Mais fácil de gerenciar no ciclo de vida. | Requires precise command order. / Requer ordem precisa de comandos. |

## 🛡️ IaC Security Features | Recursos de Segurança em IaC

A IaC não apenas cria recursos; ela os protege através de mecanismos automáticos:

1. **Drift Detection (Detecção de Desvio):**
* 🇺🇸 *Identifies when resources change from their original secure state over time.*
* 🇧🇷 Identifica quando os recursos se desviam de seu estado seguro original ao longo do tempo.


2. **Source Control (Controle de Origem/Versão):**
* 🇺🇸 *Tracks every change and allows immediate rollback if a security flaw is introduced.*
* 🇧🇷 Rastreia cada mudança e permite rollback imediato se uma falha de segurança for introduzida.


3. **Security Scanning (Escaneamento de Segurança):**
* 🇺🇸 *Tools like **Terrascan, Checkov, and TFLint** scan templates for vulnerabilities before deployment.*
* 🇧🇷 Ferramentas como **Terrascan, Checkov e TFLint** escaneiam templates em busca de vulnerabilidades antes da implementação.

## 🛡️ Analyst's Insight | Insight do Analista

**🇺🇸 English:**
The mention of **IaC Templates** is critical. While they save time, they can be "Trojan Horses." A pre-made template might have an open SSH port (22) or a public S3 bucket by default. As an analyst, you must integrate **Control Gates** in your **CI/CD pipeline** to stop any template that doesn't pass a security scan.

**🇧🇷 Português:**
*A menção a **Templates de IaC** é crítica. Embora economizem tempo, podem ser "Cavalos de Troia". Um template pronto pode vir com uma porta SSH (22) aberta ou um bucket S3 público por padrão. Como analista, você deve integrar **Control Gates (Portões de Controle)** no seu **pipeline de CI/CD** para barrar qualquer template que não passe no escaneamento de segurança.*

---

### 🚀 Key Takeaway

**"Declarative tools focus on the destination; Imperative tools focus on the journey."**
*(Ferramentas declarativas focam no destino; ferramentas imperativas focam na jornada.)*
