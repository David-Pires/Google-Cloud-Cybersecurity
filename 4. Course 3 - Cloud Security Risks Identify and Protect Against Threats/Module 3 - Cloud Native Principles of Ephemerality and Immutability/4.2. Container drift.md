

# 🛡️ Container Drift & SBOM | Desvio e Governança

### 🇺🇸 English

**Container Drift** occurs when a running container changes from its original image due to writes in its **writable layer**. This makes initial security scans obsolete and can hide malware like crypto-miners or backdoors. To combat this, we use **Immutability** and tools like **SBOM** to track every "ingredient" in our software.

### 🇧🇷 Português

*O **Desvio de Container (Container Drift)** ocorre quando um container em execução se altera em relação à sua imagem original devido a gravações em sua **camada de escrita**. Isso torna os escaneamentos de segurança iniciais obsoletos e pode esconder malwares como mineradores de criptomoeda ou backdoors. Para combater isso, usamos a **Imutabilidade** e ferramentas como **SBOM** para rastrear cada "ingrediente" do nosso software.*

---

## ⚠️ The Danger of Writable Layers | O Perigo das Camadas de Escrita

Embora a camada de escrita ofereça flexibilidade, ela abre portas para:

* **Malware Persistence:** 🇺🇸 *Attackers can download and execute malicious scripts directly into the running container.* / 🇧🇷 Atacantes podem baixar e executar scripts maliciosos diretamente no container em execução.
* **Shadow Vulnerabilities:** 🇺🇸 *New vulnerabilities are introduced that weren't present in the original scanned image.* / 🇧🇷 Novas vulnerabilidades são introduzidas, as quais não estavam presentes na imagem original escaneada.
* **Increased Costs:** 🇺🇸 *Unauthorized crypto-miners consume cloud CPU and RAM, driving up operational costs.* / 🇧🇷 Mineradores não autorizados consomem CPU e RAM da nuvem, aumentando os custos operacionais.

---

## 🛡️ Defenses: Immutability & Drift Prevention

1. **Drift Prevention (Prevenção de Desvio):**
* 🇺🇸 *Security tools that block any execution of files that were not part of the original, immutable image.*
* 🇧🇷 Ferramentas de segurança que bloqueiam a execução de qualquer arquivo que não fizesse parte da imagem original e imutável.


2. **Immutability Strategy:**
* 🇺🇸 *If you need to update code, **don't patch the container**. Build a new image, test it, and redeploy.*
* 🇧🇷 Se precisar atualizar o código, **não corrija o container**. Construa uma nova imagem, teste-a e faça o novo deploy.



---

## 📋 SBOM: The Software Bill of Materials

Pense no **SBOM** como o "rótulo de ingredientes" de um produto alimentício, mas para software.

* **🇺🇸 Definition:** A machine-readable inventory of all software components, third-party libraries, and dependencies.
* **🇧🇷 Definição:** Um inventário legível por máquina de todos os componentes de software, bibliotecas de terceiros e dependências.
* **Standard (SPDX):** 🇺🇸 *Software Package Data Exchange is the common standard for SBOMs.* / 🇧🇷 O SPDX é o padrão comum para SBOMs.

**What an SBOM tracks:**

* Package and file information.
* Relationships between elements.
* Licensing information.
* Security vulnerabilities in specific versions of libraries.

---

## 🛡️ Analyst's Insight | Insight do Analista

**🇺🇸 English:**
In a real-world incident response, the **SBOM** is your best friend. If a new vulnerability (Zero-day) is discovered in a popular library (like Log4j), you don't need to manually check every server. You search your SBOM database to immediately identify which containers are using that specific version and need to be replaced.

**🇧🇷 Português:**
*Em uma resposta a incidentes no mundo real, o **SBOM** é seu melhor amigo. Se uma nova vulnerabilidade (Zero-day) for descoberta em uma biblioteca popular (como o Log4j), você não precisa verificar manualmente cada servidor. Você pesquisa no seu banco de dados SBOM para identificar imediatamente quais containers estão usando aquela versão específica e precisam ser substituídos.*

---

### 🚀 Key Takeaway

**"If a container can change at runtime, it can't be trusted. Enforce drift prevention to keep it immutable."**
*(Se um container pode mudar em tempo de execução, ele não é confiável. Force a prevenção de desvio para mantê-lo imutável.)*
