# 🤝 Shared Responsibility & Access Control | Responsabilidade Compartilhada e Controle de Acesso

### 🇺🇸 English

The **Shared Responsibility Model** is the foundation of cloud security. It defines who is responsible for what. A common way to remember this is: the CSP is responsible for the security **of** the cloud (hardware, data centers), while the customer is responsible for security **in** the cloud (data, configurations, IAM).

### 🇧🇷 Português

*O **Modelo de Responsabilidade Compartilhada** é a base da segurança em nuvem. Ele define quem é responsável pelo quê. Uma forma comum de lembrar é: o CSP é responsável pela segurança **da** nuvem (hardware, data centers), enquanto o cliente é responsável pela segurança **na** nuvem (dados, configurações, IAM).*

## 🏗️ The Pillars of Cloud Governance | Pilares da Governança

### 1. Misconfiguration: The "Rusty Lock" Risk

* **🇺🇸 English:** Even the strongest infrastructure can be compromised by a **misconfiguration**. Like building a high-tech fence but using a "rusty lock," an improperly configured security group or S3 bucket can leave data exposed. It is the professional's duty to stay updated on the latest threats.
* **🇧🇷 Português:** Até a infraestrutura mais forte pode ser comprometida por uma **falha de configuração (misconfiguration)**. Como construir uma cerca de alta tecnologia, mas usar um "cadeado enferrujado", um grupo de segurança ou bucket S3 mal configurado pode deixar dados expostos. É dever do profissional manter-se atualizado sobre as últimas ameaças.

### 2. Multilevel Security & Access Control

* **🇺🇸 English:** You must manage access based on **data sensitivity and classification**. Using **Identity and Access Management (IAM)**, you assign permissions so that only the right people have access to the right information level.
* **🇧🇷 Português:** Você deve gerenciar o acesso com base na **sensibilidade e classificação dos dados**. Usando o **IAM (Gestão de Identidade e Acesso)**, você atribui permissões para que apenas as pessoas certas tenham acesso ao nível de informação correto.

## 🛡️ Analyst's Insight | Insight do Analista

**🇺🇸 English:**
In the cloud, "Security by Design" means assuming the physical layer is safe but the logical layer is your battlefield. Most breaches today are not due to CSP failures, but due to **customers** misconfiguring their IAM policies or leaving storage public. Always audit your configurations!

**🇧🇷 Português:**
*Na nuvem, "Segurança por Design" significa assumir que a camada física está segura, mas a camada lógica é o seu campo de batalha. A maioria das invasões hoje não se deve a falhas do CSP, mas a **clientes** configurando mal suas políticas de IAM ou deixando armazenamentos públicos. Sempre audite suas configurações!*
