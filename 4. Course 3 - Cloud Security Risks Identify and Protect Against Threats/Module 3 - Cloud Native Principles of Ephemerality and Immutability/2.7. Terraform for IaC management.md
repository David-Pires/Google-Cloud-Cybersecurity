

# 🏗️ Terraform & The Secure Workflow | Terraform e o Fluxo Seguro

### 🇺🇸 English

**Terraform** is a declarative IaC tool that allows you to provision resources by communicating with a Cloud Provider's **API**. Its strength lies in **Reproducibility** (creating the same environment multiple times) and **Consistency** (ensuring dev, staging, and prod environments are identical).

### 🇧🇷 Português

*O **Terraform** é uma ferramenta de IaC declarativa que permite provisionar recursos comunicando-se com a **API** de um provedor de nuvem. Sua força reside na **Reprodutibilidade** (criar o mesmo ambiente várias vezes) e na **Consistência** (garantir que os ambientes de desenvolvimento, homologação e produção sejam idênticos).*

---

## 🔄 The 3-Step Core Workflow | O Fluxo de 3 Passos

O Terraform segue um processo lógico para evitar erros humanos:

1. **Write (Escrever):**
* 🇺🇸 *You define the resources in configuration files (HCL).*
* 🇧🇷 Você define os recursos em arquivos de configuração (HCL).
* *Ex: Definindo uma VM Google Compute F1-micro.*


2. **Plan (Planejar):**
* 🇺🇸 *Run `terraform plan` to preview changes. This is a feedback loop to ensure the design is error-free before spending money or creating risks.*
* 🇧🇷 Execute `terraform plan` para pré-visualizar as mudanças. É um ciclo de feedback para garantir que o design esteja livre de erros antes de gastar dinheiro ou criar riscos.


3. **Apply (Aplicar):**
* 🇺🇸 *Run `terraform apply`. Terraform talks to the API and triggers the creation, modification, or deletion of resources.*
* 🇧🇷 Execute `terraform apply`. O Terraform fala com a API e aciona a criação, modificação ou deleção dos recursos.



---

## 🛡️ Sentinel: Policy as Code (PaC)

Um dos pontos mais importantes para sua carreira de segurança é onde a política entra no fluxo:

* **🇺🇸 English:** **Sentinel** is a functional policy-as-code framework. It sits **between Plan and Apply**. It acts as a "security gate." If the Terraform plan violates a policy (e.g., "The VM must have encryption enabled"), Sentinel blocks the `apply` step.
* **🇧🇷 Português:** **Sentinel** é um framework de política como código. Ele fica **entre o Plan e o Apply**. Ele age como um "portão de segurança". Se o plano do Terraform violar uma política (ex: "A VM deve ter criptografia habilitada"), o Sentinel bloqueia o passo de `apply`.

## 🛡️ Analyst's Insight | Insight do Analista

**🇺🇸 English:**
Terraform is a "Stateful" tool. It keeps a `terraform.tfstate` file that acts as the "Source of Truth." As a security professional, protecting this state file is critical because it contains the mapping of your entire infrastructure and potentially sensitive metadata. Always store it in a secure, encrypted remote backend (like a Google Cloud Storage bucket with restricted access).

**🇧🇷 Português:**
*O Terraform é uma ferramenta "Stateful" (com estado). Ele mantém um arquivo `terraform.tfstate` que age como a "Fonte da Verdade". Como profissional de segurança, proteger esse arquivo de estado é crítico, pois ele contém o mapeamento de toda a sua infraestrutura e metadados potencialmente sensíveis. Sempre o armazene em um backend remoto seguro e criptografado (como um bucket do Google Cloud Storage com acesso restrito).*

### 🚀 Key Takeaway

**"Plan before you Apply, and use Sentinel to ensure you Comply."**
*(Planeje antes de Aplicar, e use o Sentinel para garantir que você esteja em Conformidade.)*
