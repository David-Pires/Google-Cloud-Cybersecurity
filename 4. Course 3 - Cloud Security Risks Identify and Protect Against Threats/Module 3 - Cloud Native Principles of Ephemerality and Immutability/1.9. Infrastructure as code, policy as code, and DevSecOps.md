
# 📜 Infrastructure & Policy as Code (IaC & PaC)

### 🇺🇸 English

The shift to cloud-native security means moving from manual configurations to **Infrastructure as Code (IaC)** and **Policy as Code (PaC)**. By using APIs, security professionals can define the entire environment and its rules in text files, allowing for version control and automated enforcement.

### 🇧🇷 Português

*A mudança para a segurança nativa da nuvem significa passar de configurações manuais para **Infraestrutura como Código (IaC)** e **Política como Código (PaC)**. Ao usar APIs, os profissionais de segurança podem definir todo o ambiente e suas regras em arquivos de texto, permitindo controle de versão e aplicação automatizada.*

---

## 🏗️ IaC: Declarative vs. Imperative

Existem duas formas de dizer à nuvem o que fazer:

| Approach / Abordagem | 🇺🇸 Focus | 🇧🇷 Foco |
| --- | --- | --- |
| **Declarative (Declarativa)** | **What:** You define the final state (e.g., "I want 3 servers"). | **O quê:** Você define o estado final (ex: "Quero 3 servidores"). |
| **Imperative (Imperativa)** | **How:** You define specific steps (e.g., "Create A, then B, then C"). | **Como:** Você define passos específicos (ex: "Crie A, depois B, depois C"). |

---

## ⚖️ Policy as Code (PaC)

* **🇺🇸 English:** PaC uses high-level code to automate rules and conditions. Instead of a PDF manual, security policies are scripts that run automatically to block non-compliant resources (e.g., "No database can be created without encryption").
* **🇧🇷 Português:** PaC usa código de alto nível para automatizar regras e condições. Em vez de um manual em PDF, as políticas de segurança são scripts que rodam automaticamente para bloquear recursos fora de conformidade (ex: "Nenhum banco de dados pode ser criado sem criptografia").

## 🚀 DevSecOps: Security at the Speed of Code

### 🇺🇸 English

**DevSecOps** is the integration of security into the **DevOps** workflow. Its goal is to identify vulnerabilities early (Shift Left) through collaboration between developers, operations, and security teams.

### 🇧🇷 Português

***DevSecOps*** é a integração da segurança no fluxo de trabalho **DevOps**. Seu objetivo é identificar vulnerabilidades cedo (*Shift Left*) através da colaboração entre desenvolvedores, operações e equipes de segurança.

**DevSecOps relies on:**

* **CI/CD Pipelines:** Continuous integration and deployment.
* **Version Control:** Tracking changes in code (e.g., Git).
* **Continuous Testing & Monitoring:** Checking for threats and errors in real-time.

---

## 🛡️ Analyst's Insight | Insight do Analista

**🇺🇸 English:**
In the DevSecOps model, security is no longer a "bottleneck" at the end of the project. By using **Policy as Code**, you empower developers to build securely from day one. If their code violates a security policy, the **CI/CD pipeline** fails automatically, giving them instant feedback to fix the issue.

**🇧🇷 Português:**
*No modelo DevSecOps, a segurança não é mais um "gargalo" no final do projeto. Ao usar **Política como Código**, você capacita os desenvolvedores a construir de forma segura desde o primeiro dia. Se o código deles violar uma política de segurança, o **pipeline de CI/CD** falha automaticamente, dando feedback instantâneo para corrigirem o problema.*
