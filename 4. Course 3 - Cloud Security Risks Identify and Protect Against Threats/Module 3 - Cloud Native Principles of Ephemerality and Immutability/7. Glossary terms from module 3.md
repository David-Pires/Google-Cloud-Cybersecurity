

## 🏗️ Design e Arquitetura (Cloud-Native)

* **Cloud-native design:** O método de criar aplicações otimizadas para a nuvem.
* **Ephemerality (Efemeridade):** Conceito de que as coisas existem por pouco tempo. Na segurança, isso limita o tempo que um invasor tem para explorar um recurso.
* **Immutability (Imutabilidade):** Incapacidade de alterar um objeto após sua criação. Se algo precisa mudar, você cria um novo. Isso previne modificações maliciosas em tempo de execução.
* **Shared responsibility model (Modelo de Responsabilidade Compartilhada):** O acordo que define quem cuida de quê (o provedor cuida da infraestrutura física, você cuida dos dados e do código).

---

## 🤖 Automação e Governança

* **Automation (Automação):** Uso da tecnologia para reduzir o esforço humano em tarefas repetitivas.
* **Infrastructure as code (IaC):** Prática de gerenciar infraestrutura (redes, servidores) usando scripts reutilizáveis (ex: Terraform).
* **Policy as code (PaC):** Uso de código para definir e automatizar regras de segurança e conformidade.
* **Software bill of materials (SBOM):** Uma lista (estilo rótulo de ingredientes) de todos os componentes e bibliotecas de um software. Essencial para rastrear vulnerabilidades na cadeia de suprimentos.

---

## 📦 Virtualização e Containers

* **Container:** Pacote de software com apenas o necessário para rodar uma aplicação. É leve e portátil.
* **Hypervisor:** Camada de abstração que permite criar Máquinas Virtuais (VMs) sobre o hardware físico.
* **Kernel:** O "cérebro" do sistema operacional que gerencia processos e memória. Containers compartilham o Kernel do hospedeiro.
* **Container layer:** Espaço de gravação (camada de escrita) dentro de um container. É onde ocorre o perigoso "Container Drift".
* **Container runtimes:** O software (ex: Docker) que realmente executa e gerencia os containers no sistema.

---

## ☸️ Orquestração (Kubernetes)

* **Container clusters:** Sistemas que gerenciam containers agrupados em "Pods".
* **Application Programming Interface (API):** O ponto de comunicação entre aplicações. No Kubernetes, tudo é feito via chamadas de API.
* **Access controls:** Controles que gerenciam quem pode acessar o quê, garantindo autorização e prestação de contas.
* **Rate limiting:** Método para limitar a frequência de operações, prevenindo abusos ou ataques de negação de serviço (DoS).
* **Managed service:** Um serviço ou ecossistema gerenciado por terceiros (ex: GKE no Google Cloud), onde o provedor cuida da manutenção complexa.

## 🛡️ Insight do Analista

**🇺🇸 English:**
When you combine **Immutability** with **SBOM**, you create a "Verifiable Infrastructure." If a new zero-day vulnerability appears, you check your SBOM to see which containers are affected and use **IaC** to redeploy the entire environment with patched versions in minutes. This is the gold standard of modern security.

**🇧🇷 Português:**
*Quando você combina **Imutabilidade** com **SBOM**, você cria uma "Infraestrutura Verificável". Se uma nova vulnerabilidade zero-day aparece, você checa seu SBOM para ver quais containers são afetados e usa **IaC** para fazer o redeploy de todo o ambiente com versões corrigidas em minutos. Este é o padrão ouro da segurança moderna.*
