### 📄 Exemplo de Terraform: Bucket Seguro (GCS)

```hcl
# 1. Definindo o recurso do Bucket
resource "google_storage_bucket" "secure_assets" {
  name          = "meu-bucket-seguro-2026"
  location      = "US"
  force_destroy = true

  # 2. Prevenindo acesso público acidental (Segurança Postural)
  public_access_prevention = "enforced"

  # 3. Criptografia em repouso (Encryption at Rest)
  encryption {
    default_kms_key_name = "minha-chave-de-criptografia-mestra"
  }

  # 4. Versionamento (Proteção contra deleção acidental)
  versioning {
    enabled = true
  }
}

```

---

### 🛡️ Por que isso é mais seguro que o método manual?

### 🇺🇸 English

1. **Peer Review:** Before this bucket is created, another security engineer can review the code to ensure the "public_access_prevention" is set to "enforced".
2. **Audit Trail:** Every change to the bucket is recorded in your version control system (like GitHub). You know *who* changed *what* and *when*.
3. **Consistency:** If you need to create 10 identical secure buckets, the code ensures they will all have the exact same security settings.

### 🇧🇷 Português

1. **Revisão por Pares:** Antes que este bucket seja criado, outro engenheiro de segurança pode revisar o código para garantir que a prevenção de acesso público esteja ativada.
2. **Trilha de Auditoria:** Cada mudança no bucket é registrada no seu sistema de controle de versão (como o GitHub). Você sabe *quem* mudou, *o quê* e *quando*.
3. **Consistência:** Se você precisar criar 10 buckets seguros idênticos, o código garante que todos terão exatamente as mesmas configurações de segurança.

### ⚙️ Como a Automação se integra com a Segurança (DevSecOps)

O grande segredo da automação em nuvem é o pipeline de **CI/CD**. Nele, o código de infraestrutura passa por testes automáticos de segurança antes de chegar à nuvem.

1. **Linting/Static Analysis:** Ferramentas automáticas escaneiam seu código Terraform em busca de erros (ex: "Você esqueceu de habilitar a criptografia!").
2. **Terraform Plan:** O sistema mostra exatamente o que será criado antes de executar, evitando surpresas.
3. **Deployment:** O sistema provisiona os recursos de forma idêntica ao que foi testado.

### 🛡️ Analyst's Insight | Insight do Analista

**🇺🇸 English:**
This is "Policy as Code". You are not just building a bucket; you are codifying a security policy. If a manual change occurs in the Cloud Console (Configuration Drift), Terraform can detect it and revert the resource to its original secure state.

**🇧🇷 Português:**
*Isso é "Política como Código". Você não está apenas construindo um bucket; você está codificando uma política de segurança. Se ocorrer uma alteração manual no Console da Nuvem (Desvio de Configuração), o Terraform pode detectá-la e reverter o recurso para o seu estado seguro original.*
