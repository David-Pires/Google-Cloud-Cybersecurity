
# 📖 Glossário Revisado: Módulo 2 | Unit 2 Official Glossary

### 🛡️ Estratégias e Frameworks de Ameaças

* **🇺🇸 Advanced Persistent Threat (APT) | 🇧🇷 Ameaça Persistente Avançada:** Um adversário com níveis sofisticados de expertise e recursos significativos, que alcança seus objetivos através de múltiplos vetores de ataque.
* **🇺🇸 MITRE ATT&CK®:** Um framework (matriz) global utilizado para compreender e abordar ameaças com base em observações do mundo real.
* **🇺🇸 Tactics | 🇧🇷 Táticas:** O motivo/razão de um agente malicioso para realizar uma ação ou técnica (o "porquê").
* **🇺🇸 Techniques | 🇧🇷 Técnicas:** As ações específicas que um agente malicioso utilizou para alcançar seu objetivo (o "como").
* **🇺🇸 Procedures | 🇧🇷 Procedimentos:** A implementação detalhada e específica de uma técnica (o "passo a passo").
* **🇺🇸 Indicators of Compromise (IoC) | 🇧🇷 Indicadores de Comprometimento:** Evidências observáveis que sugerem sinais de um potencial incidente de segurança.

### 🔍 Operações e Monitoramento

* **🇺🇸 Security Monitoring | 🇧🇷 Monitoramento de Segurança:** Processo sistemático de vigilância de sistemas para detectar e lidar com potenciais violações ou incidentes de segurança.
* **🇺🇸 Threat Hunting | 🇧🇷 Caça a Ameaças:** Método proativo de identificar ameaças anteriormente desconhecidas dentro de uma rede.
* **🇺🇸 False Positive | 🇧🇷 Falso Positivo:** Um alerta que detecta incorretamente a presença de uma ameaça quando a atividade é legítima.

### 📊 Processamento e Análise de Dados

* **🇺🇸 Aggregation | 🇧🇷 Agregação:** O processo de coletar e consolidar diversas formas de dados em um local centralizado.
* **🇺🇸 Correlation | 🇧🇷 Correlação:** O relacionamento e a conexão entre dois ou mais eventos de segurança.
* **🇺🇸 Unified Data Model (UDM) | 🇧🇷 Modelo de Dados Unificado:** Um modelo de dados (usado pelo Chronicle) para processar e armazenar dados de forma consistente.
* **🇺🇸 Regular Expression (RegEx) | 🇧🇷 Expressão Regular:** Uma sequência de caracteres que forma um padrão de busca em textos e logs.
* **🇺🇸 YARA-L:** Uma linguagem de computador (sintaxe do Chronicle) usada para criar regras de detecção para pesquisar dados de log ingeridos.

## 🛡️ Insight do Analista (Conexão Técnica)

**🇺🇸 English:**
In a practical scenario, an analyst uses **YARA-L** to write a rule based on **UDM** fields. This rule might search for **Techniques** like "Brute Force" by identifying **IoCs** (failed login patterns) through **Correlation** of multiple auth logs. If the rule is too broad, it triggers a **False Positive**.

**🇧🇷 Português:**
*Em um cenário prático, um analista usa **YARA-L** para escrever uma regra baseada em campos **UDM**. Esta regra pode buscar por **Técnicas** como "Brute Force" ao identificar **IoCs** (padrões de login falhos) através da **Correlação** de múltiplos logs de autenticação. Se a regra for muito ampla, ela dispara um **Falso Positivo**.*
