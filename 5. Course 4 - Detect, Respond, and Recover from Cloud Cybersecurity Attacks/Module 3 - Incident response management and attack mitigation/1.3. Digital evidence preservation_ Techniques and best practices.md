

# 📑 Técnicas e Melhores Práticas de Preservação | Preservation Techniques

**🇺🇸 English:** Digital evidence is volatile and fragile. Proper preservation involves using bit-by-bit duplication and cryptographic hashes to prove that the data analyzed by the **team** is an exact, unaltered copy of the original artifact.
**🇧🇷 Português:** A evidência digital é volátil e frágil. A preservação adequada envolve o uso de duplicação bit a bit e hashes criptográficos para provar que os dados analisados pela **equipe** são uma cópia exata e inalterada do artefato original.

---

### 1. Técnicas de Preservação de Integridade | Integrity Techniques

Para que uma evidência seja válida, ela deve passar por processos que garantam a sua não alteração:

* **🇺🇸 Disk Imaging | 🇧🇷 Imagem de Disco:**
* 🇺🇸 *Creation of a bit-by-bit duplication of the evidence file.*
* 🇧🇷 *Criação de uma duplicação bit a bit do arquivo de evidência.*


* **🇺🇸 Write-Blocking | 🇧🇷 Bloqueio de Escrita:**
* 🇺🇸 *Techniques that prevent any modification to the original device during analysis.*
* 🇧🇷 *Técnicas que impedem qualquer modificação no dispositivo original durante a análise.*


* **🇺🇸 Cryptographic Hashes | 🇧🇷 Hashes Criptográficos:**
* 🇺🇸 *A digital checksum. If a single bit in the file changes, the hash changes, signaling a loss of integrity.*
* 🇧🇷 *Um "checksum" digital. Se um único bit no arquivo mudar, o hash muda, sinalizando perda de integridade.*



---

### 2. Documentação e Cadeia de Custódia | Documentation & Chain of Custody

**🇺🇸 English:** A clear **Chain of Custody** establishes a timeline for handling evidence, documenting names, titles, and reasons for every transfer.
**🇧🇷 Português:** Uma **Cadeia de Custódia** clara estabelece uma linha do tempo para o manuseio da evidência, documentando nomes, cargos e motivos para cada transferência.

**Perguntas Fundamentais na Coleta | Core Investigative Questions:**

1. **Who (Quem):** Quem esteve envolvido no crime cibernético?
2. **What (O quê):** O que aconteceu exatamente?
3. **When (Quando):** Qual foi a cronologia dos eventos?
4. **Where (Onde):** Onde os dados estavam localizados ou para onde foram?
5. **How (Como):** Como a vulnerabilidade foi explorada?

---

### 3. Considerações de Nuvem (Cloud-Based IR) | Cloud Considerations

**🇺🇸 English:** Storing evidence in the cloud offers resilient storage but requires additional security layers, such as Multi-Factor Authentication (MFA) and pre-storage encryption.
**🇧🇷 Português:** Armazenar evidências na nuvem oferece um armazenamento resiliente, mas exige camadas adicionais de segurança, como Autenticação de Múltiplos Fatores (MFA) e criptografia antes do armazenamento.

* **🇺🇸 Encryption Key Management:** Ensure keys are retrievable for future investigations.
* **🇧🇷 Gestão de Chaves de Criptografia:** Garantir que as chaves sejam recuperáveis para investigações futuras.
* **🇺🇸 Provider Strategy:** Have a plan to move data if the cloud provider changes their services.
* **🇧🇷 Estratégia de Provedor:** Ter um plano para mover os dados caso o provedor de nuvem altere seus serviços.

## 🛡️ Insight do Analista | Analyst's Insight

**🇺🇸 English:**
In the cloud, you don't physically "seize" a hard drive. You seize a **Snapshot**. The use of **Cryptographic Hashes** is even more vital here because the storage is virtual. By generating a hash immediately after the snapshot, the **team** creates a digital seal that guarantees the forensic copy hasn't been modified by cloud administrators or automated scripts.

**🇧🇷 Português:**
*Na nuvem, você não "apreende" fisicamente um disco rígido. Você apreende um **Snapshot**. O uso de **Hashes Criptográficos** é ainda mais vital aqui porque o armazenamento é virtual. Ao gerar um hash imediatamente após o snapshot, a **equipe** cria um selo digital que garante que a cópia forense não foi modificada por administradores de nuvem ou scripts automatizados.*

### 🚀 Glossário Técnico | Technical Glossary

* **🇺🇸 Bit-by-bit duplication:** A copy process that includes every single bit of data, including deleted files and unallocated space.
* **🇧🇷 Duplicação bit a bit:** Um processo de cópia que inclui cada bit de dados, incluindo arquivos deletados e espaço não alocado.
* **🇺🇸 Write-blocking:** Hardware or software that allows read-only access to data storage without the risk of accidental writes.
* **🇧🇷 Bloqueio de escrita:** Hardware ou software que permite acesso apenas de leitura ao armazenamento de dados sem o risco de gravações acidentais.
* **🇺🇸 Volatile Evidence:** Data that is lost when a device is powered off (e.g., RAM data).
* **🇧🇷 Evidência Volátil:** Dados que são perdidos quando um dispositivo é desligado (ex: dados na memória RAM).

**"In forensics, if it isn't documented and hashed, it didn't happen."**
*(Em forense, se não foi documentado e não possui um hash, não aconteceu.)*
