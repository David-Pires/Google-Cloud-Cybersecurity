
# 🧠 Metodologias de Detecção: Assinaturas vs. Anomalias

Para proteger uma rede, os sistemas de detecção precisam de critérios para decidir o que é uma ameaça. Eles utilizam dois métodos principais que se complementam.

---

### 1. Detecção Baseada em Assinaturas | Signature-based Detection

**🇺🇸 English:** A signature is a specific pattern associated with known malicious activity. The system checks data packets for factors like source/destination IPs, ports, protocols, and payload details, comparing them against a database of known threats.
**🇧🇷 Português:** Uma assinatura é um padrão específico associado a atividades maliciosas conhecidas. O sistema verifica pacotes de dados em busca de fatores como IPs de origem/destino, portas, protocolos e detalhes do payload, comparando-os com um banco de dados de ameaças conhecidas.

* **Vantagem:** Muito eficaz contra ataques conhecidos; baixos índices de falsos positivos.
* **Limitação:** Ineficaz contra ataques novos (**Zero-day**) que ainda não possuem assinatura.
* **Exemplo Técnico (Python):**
```python
# Exemplo de uma assinatura simplificada
signature = {
    "source_ip": "192.168.1.50",
    "protocol": "TCP",
    "dest_port": 80,
    "payload": "exploit_payload"
}

```



---

### 2. Detecção Baseada em Anomalias | Anomaly-based Detection

**🇺🇸 English:** This method involves creating a **normal baseline** of network activity. The system acts as a "tireless guard," alerting whenever current traffic significantly deviates from this established standard.
**🇧🇷 Português:** Este método envolve a criação de uma **linha de base (baseline)** do que é considerado atividade normal. O sistema atua como um "guarda incansável", alertando sempre que o tráfego atual se desvia significativamente desse padrão estabelecido.

* **Vantagem:** Consegue detectar ameaças novas e desconhecidas (Zero-day) que não possuem assinaturas.
* **Limitação:** Pode gerar mais falsos positivos se o comportamento normal da rede mudar legitimamente.
* **Exemplo Técnico (SQL):** Utilizado para modelar o comportamento histórico e identificar desvios.
```sql
-- Criando uma baseline histórica
SELECT source_ip, dest_port, protocol, COUNT(*) as activity_count
FROM network_logs
WHERE timestamp > current_date - 30
GROUP BY source_ip, dest_port, protocol;

```



---

### ⚖️ Comparativo Direto | Quick Comparison

| 🔍 Critério | 🇺🇸 Signature-based | 🇧🇷 Baseada em Assinaturas | 🇺🇸 Anomaly-based | 🇧🇷 Baseada em Anomalias |
| --- | --- | --- | --- | --- |
| **Lógica** | "Known Bad" | "Mal conhecido" | "Not Normal" | "Não normal" |
| **Referência** | Database / Assinaturas | Banco de Dados | Baseline / Histórico | Linha de Base |
| **Ponto Forte** | Speed & Accuracy | Velocidade e Precisão | Zero-day detection | Detecta ataques novos |
| **Ponto Fraco** | Misses new threats | Ignora ameaças novas | High false positives | Muitos falsos positivos |

## 🛡️ Insight do Analista | Analyst's Insight

**🇺🇸 English:**
Think of **Signature-based** detection like a "Wanted" poster: it only catches the criminals it already knows. **Anomaly-based** detection is like a security guard who knows everyone in the building; if a stranger walks in or someone starts acting weirdly, he notices immediately, even if he doesn't know their name. In SecOps, you need both.

**🇧🇷 Português:**
*Pense na detecção **Baseada em Assinaturas** como um cartaz de "Procurado": ela só pega os criminosos que já conhece. A detecção **Baseada em Anomalias** é como um segurança que conhece todos no prédio; se um estranho entra ou alguém começa a agir de forma estranha, ele percebe imediatamente, mesmo que não saiba o nome da pessoa. No SecOps, você precisa de ambos.*

### 🚀 Resumo de Termos Chave | Key Terms Summary

* **Baseline (Linha de Base):** O padrão de tráfego "saudável" ou normal de uma rede.
* **Payload:** A carga útil ou conteúdo de dados de um pacote de rede.
* **False Positive (Falso Positivo):** Quando o sistema identifica erroneamente uma atividade legítima como maliciosa.
* **Zero-day:** Uma vulnerabilidade ou ataque que é desconhecido para os defensores e fabricantes.

**"Signatures detect the past; anomalies detect the future."**
*(Assinaturas detectam o passado; anomalias detectam o futuro.)*
