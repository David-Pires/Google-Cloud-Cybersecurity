

# 📑 Tipos de Logs e Gestão de Registos | Log Types & Management

**🇺🇸 English:** Logs are records of events that provide transparency and context for everything that happens in a cloud environment. Log management is the process of collecting, storing, and analyzing this data to ensure security and compliance.
**🇧🇷 Português:** Logs são registros de eventos que fornecem transparência e contexto para tudo o que acontece num ambiente de nuvem. A gestão de logs é o processo de coletar, armazenar e analisar esses dados para garantir a segurança e a conformidade.

### 2. Logs de Identidade e Aplicação | Identity & Application Logs

Para além da infraestrutura, é necessário monitorizar os utilizadores e o software.

#### **I. Logs de Identidade | Identity Logs**

* **Admin Logs:**
* 🇺🇸 *Actions by administrators, such as adding or removing users.*
* 🇧🇷 *Ações realizadas por administradores, como adicionar ou remover utilizadores.*


* **User Logs:**
* 🇺🇸 *Actions by users, like password changes or recovery info updates.*
* 🇧🇷 *Ações dos utilizadores, como alterações de senha ou atualizações de recuperação.*



#### **II. Logs de Aplicação | Application Logs**

* 🇺🇸 **English:** Also known as **User-written logs**, these are essential for troubleshooting and detecting if an application was compromised or if data was stolen.
* 🇧🇷 **Português:** Também conhecidos como **Logs escritos pelo utilizador**, são essenciais para resolução de problemas e para detetar se uma aplicação foi comprometida ou se dados foram roubados.

---

### 3. Gestão e Fluxo de Logs | Log Management & Flow

A gestão eficiente utiliza **Roteadores (Routers)** e **Destinos (Sinks)** para controlar o custo e a utilidade dos dados.

**🇺🇸 English:**

* **Log Routers:** Used to filter logs based on content and control how they are distributed.
* **Log Sinks:** Destinations for storage or analysis, such as **Cloud Storage** (compliance), **BigQuery** (analysis), or **Pub/Sub** (streaming to external SIEMs).

**🇧🇷 Português:**

* **Roteadores de Logs (Routers):** Usados para filtrar logs com base no conteúdo e controlar como são distribuídos.
* **Destinos de Logs (Sinks):** Destinos para armazenamento ou análise, como **Cloud Storage** (conformidade), **BigQuery** (análise) ou **Pub/Sub** (envio para SIEMs externos).

## 🛡️ Insight do Analista | Analyst's Insight

**🇺🇸 English:**
A critical detail: **Admin Activity** logs are ON by default and free, but **Data Access** logs are OFF by default because they generate massive volumes of data. You must manually enable them for sensitive services, but watch your budget—storage costs can escalate quickly if retention periods are not optimized.

**🇧🇷 Português:**
*Um detalhe crítico: os logs de **Atividade de Admin** estão ativos por padrão e são gratuitos, mas os logs de **Acesso a Dados** estão desativados por padrão porque geram volumes massivos de dados. Deve ativá-los manualmente para serviços sensíveis, mas atenção ao orçamento — os custos de armazenamento podem subir rapidamente se os períodos de retenção não forem otimizados.*

### 🚀 Glossário Técnico | Technical Glossary

* **🇺🇸 Log Analysis:** The process of examining logs to identify significant events.
* **🇧🇷 Análise de Logs:** O processo de examinar logs para identificar eventos significativos.
* **🇺🇸 Log Sink:** A destination where logs are routed for long-term storage or processing.
* **🇧🇷 Destino de Log (Sink):** Um destino para onde os logs são roteados para armazenamento ou processamento de longo prazo.
* **🇺🇸 Data Segregation:** Separating logs from different environments (e.g., Production vs. Dev) for better security.
* **🇧🇷 Segregação de Dados:** Separar logs de diferentes ambientes (ex: Produção vs. Desenvolvimento) para maior segurança.

**"Effective logging isn't just about collecting everything; it's about routing the right data to the right place for the right amount of time."**
*(Uma monitorização eficaz não se trata apenas de recolher tudo; trata-se de rotear os dados certos para o lugar certo pelo tempo necessário.)*
