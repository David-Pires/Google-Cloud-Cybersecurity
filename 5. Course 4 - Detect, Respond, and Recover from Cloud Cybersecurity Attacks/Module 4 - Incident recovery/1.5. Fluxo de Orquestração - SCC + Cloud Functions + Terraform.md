

### 🔄 Fluxo de Orquestração: SCC + Cloud Functions + Terraform

1. **Detecção (Trigger):** O **Security Command Center** deteta um ataque de *SQL Injection* ou um acesso de um IP malicioso e gera um alerta (Finding).
2. **Encaminhamento:** Esse alerta é enviado para um tópico do **Pub/Sub** (um serviço de mensagens da GCP).
3. **Processamento (Action):** Uma **Cloud Function** (um pequeno script) é acionada pelo Pub/Sub. Ela lê o endereço IP do atacante no alerta.
4. **Execução (Flow):** A Cloud Function utiliza a **API do Terraform** ou do **Cloud Armor** para atualizar a política de segurança que criamos no passo anterior, adicionando o IP à lista de bloqueio (`deny`) automaticamente.

---

### 🛠️ Por que isto é valioso para o seu perfil Consultivo?

Como você está focado em aprender sobre **Orquestração e Automação**, este cenário demonstra o uso de um **Playbook automatizado**:

* **Redução do MTTR (Mean Time to Respond):** O bloqueio acontece em milissegundos, antes mesmo de você receber o e-mail de notificação.
* **Escalabilidade:** A mesma lógica protege centenas de aplicações simultaneamente sem intervenção humana.
* **Hardening Dinâmico:** A sua infraestrutura "aprende" com os ataques e fecha as portas em tempo real.
