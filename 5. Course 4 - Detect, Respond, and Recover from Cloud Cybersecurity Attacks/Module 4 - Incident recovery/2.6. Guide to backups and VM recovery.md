

## 🖥️ Google Cloud Backup and DR: Visão Geral

O Google Cloud Backup and DR é um serviço gerenciado que oferece uma forma centralizada de proteger dados e cargas de trabalho na nuvem e em ambientes locais.

* **Capacidades:** Permite gerenciar backups de discos do Compute Engine, bancos de dados Cloud SQL e buckets do Cloud Storage.
* **Console de Gerenciamento:** Local central para configurar atividades de backup e restauração.
* **Appliance de Backup/Recuperação:** Atua como um "movedor de dados" que captura e gerencia o ciclo de vida dos backups.

---

## 📋 O Modelo de Plano de Backup (Backup Plan Template)

Os planos de backup são regras que definem como, com que frequência e por quanto tempo os dados são retidos e replicados.

* **Políticas de Produção para Snapshot:** Definem como capturar os dados de produção de uma aplicação como um snapshot.
* **Snapshot:** Captura o estado de uma VM em um momento específico no tempo.
* **Agendamento Contínuo:** Define intervalos de tempo específicos (ex: a cada 2 horas) para a realização dos backups.

---

## 🛠️ Procedimentos Operacionais

### Como criar e aplicar um template

1. **Criar o Template:** No console, acesse *Backup Plans* > *Templates* e adicione uma política de *Snapshot* com o agendamento desejado.
2. **Aplicar à VM:** Vá em *Back Up & Recover* > *Back Up*, selecione a instância do Compute Engine e aplique o template criado.

### Como recuperar uma VM

1. Acesse *Back Up & Recover* > *Recover* e selecione a VM desejada.
2. Escolha a imagem de backup e selecione **Mount as new GCE instance** para restaurar a VM como uma nova instância.

## 🛡️ Insight para o seu Repositório (Perfil Consultivo)

A capacidade de recuperar dados rapidamente é um componente vital da resposta a incidentes, minimizando danos e suportando a continuidade operacional. Em um ambiente de nuvem dinâmico, utilizar snapshots contínuos permite que você retorne ao último estado íntegro conhecido com o menor **RPO** (Ponto de Recuperação) possível.
